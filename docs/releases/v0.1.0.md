# v0.1.0 Release Notes

Release date: 2026-06-29

这是 Map Tile Fetcher 合并为单一 Go Web 应用后的第一个开源预览版本。
本版本面向需要自托管、批量下载授权地图瓦片并生成离线产物的 GIS/地图开发者。

> 合规提醒：请只用于你有权访问和下载的地图瓦片服务。项目提供限速、重试
> 和本地 token 配置能力，但不会授权绕过任何第三方服务条款、配额或访问限制。

## 适合谁

- 需要在内网、个人服务器或项目服务器上运行地图瓦片下载工具的团队。
- 需要按 bbox、GeoJSON 或行政区划批量下载授权瓦片的人。
- 需要 ZIP 文件树或 MBTiles 产物，用于离线地图、测试数据或内部制图流程的人。

## 解决什么问题

- 把旧范围下载器和行政区划下载能力统一进一个 Go Web 应用。
- 用浏览器 UI 管理任务创建、任务状态、失败记录和产物下载。
- 用 SQLite 保存任务元数据，用文件系统保存瓦片、ZIP、MBTiles 和日志。
- 保持真实 token 在本地配置中，仓库只保留安全占位符。

## 版本重点

- 统一 Go 后端，旧 .NET 范围下载器运行代码已退休。
- 新增统一 Web UI，支持两种任务创建模式：
  - 范围框选下载
  - 行政区划/GeoJSON 下载
- 支持天地图和 Mapbox token 统一输入，两个下载模式复用同一凭证入口。
- 支持矩形 bbox 任务、放大地图选择和多边形范围选择。
- 支持立即执行和单次定时执行。
- 支持按任务选择 ZIP 文件树产物或 MBTiles 产物。
- 支持产物下载、失败记录持久化和失败瓦片重试。
- 保持登录默认启用，同时支持可信本地环境免登录。
- 新增发布预检脚本和 GitHub Actions 验证。

## 快速启动

源码运行：

```powershell
git clone https://github.com/Joe5027/map-tile-fetcher.git
cd map-tile-fetcher\apps\admin-region-tiler
go run .
```

Docker 运行：

```powershell
cd apps\admin-region-tiler
Copy-Item .env.example .env
docker compose up --build
```

打开 `http://127.0.0.1:8081/`。

开发默认账号：

- 用户名：`admin`
- 密码：`adminmap`

生产部署前请在 `.env` 中修改默认账号和密码。

## 配置说明

- `conf.toml`：地图源、输出、任务和下载区域示例配置。
- `.env.example`：端口、数据库文件名、默认登录账号示例。
- `data/`：SQLite 运行数据库目录，不能提交到 Git。
- `output/`：下载产物目录，不能提交到 Git。
- `geojson/`：可持久化的区域资源目录。

安全占位符：

- `YOUR_TIANDITU_TOKEN`
- `YOUR_MAPBOX_TOKEN`
- `YOUR_MAPBOX_SKU`

## 当前限制

- 真实地图服务的下载权限、配额和限速取决于用户自己的服务条款和 token。
- 多边形范围任务当前会生成任务专用 GeoJSON，以复用已有 tile-cover 管线。
- Linux 资产在部分解压环境中可能需要先执行 `chmod +x map-tile-fetcher`。
- GitHub Release 需要在仓库有发布权限的环境中创建并上传二进制资产。

Map Tile Fetcher v0.1.0 is the first open-source preview after the project was
merged into one Go Web app. It supports authorized map tile downloads by
bounding box or GeoJSON/admin regions, task progress, failure records, retries,
